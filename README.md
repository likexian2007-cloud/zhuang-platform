> 规范筑基 · 数字提效 ——「装配智建助力和美乡村」
>
> 校企合作 · 装配式节能结构一体化混凝土墙板技术 · 装配率 95%

本平台依据项目展演稿、PPT 与「装配式构件吊装记录」原型，并参考「匠心筑馆」指挥中心平台的布局，重构为一套**单页（SPA）数字指挥中心**：一个外壳 + 顶部模块导航，点击模块即时切换、**无整页刷新**，完整实现了材料中描述的全部功能。

**新增 · 建筑健康指数 BHI**：首页接入运维阶段智慧监测仪表盘，模拟振动、裂缝、倾角、位移及温湿度传感器实时采集，并按阈值计算 Building Health Index，自动显示“安全运行/持续关注/风险预警”；AI 运维总工可调用 DeepSeek 生成健康评估、风险原因、处置建议和复核频率，把施工 AI 巡检延伸为运维 AI 健康监测。

**新增 · AI闭环中心**：把比赛展示从“发现问题”延伸到“派单整改、规范依据、施工日报、复检销项”。平台可根据现场异常生成整改工单，自动给出责任班组、复检要点和风险等级；同时提供规范问答、风险热力图和班组评分，适合答辩时展示真实项目管理闭环。

## 二、运行方式

### 方式 A：一键启动（推荐）
双击根目录的 **`启动平台.bat`**：
- 若已安装 [Node.js](https://nodejs.org)：自动启动后端并打开 `http://localhost:8080`，**数据持久化保存**于 `backend/data/records.json`，可跨浏览器/设备共享。
- 若未安装 Node.js：自动以**离线模式**打开 `index.html`，数据保存在浏览器本地（localStorage）。

### 方式 B：直接打开
直接双击 `index.html` 即可使用全部前端功能（离线模式，纯浏览器本地存储，无需任何安装）。

### 方式 D：上线到公开网址（纯静态）
平台前端可托管到 **GitHub Pages / Netlify / Vercel / 云对象存储**，上线后数据默认存于访问者浏览器本地（localStorage）。如需线上也使用真实 DeepSeek，需要单独运行后端代理并设置 `window.PLATFORM_API`，API Key 只保存在服务端，不写入前端文件。

### 方式 C：手动启动后端
```bash
cd backend
npm start          # 等同于 node server.js
# 浏览器打开 http://localhost:8080
```
> 后端**零第三方依赖**，仅使用 Node 内置模块，无需 `npm install`。

## 三、技术栈

- **前端**：原生 HTML/CSS/JS + Tailwind(CDN) + Chart.js + jsPDF + html2canvas + qrcodejs，统一科技风深色主题（`assets/theme.css`）。
- **后端**：Node.js 内置 `http` 模块（零依赖），托管静态页面 + 质量监测记录 REST API。
- **数据**：在线模式存 `backend/data/records.json`；离线模式存浏览器 localStorage，自动降级、互不影响。

### 后端接口
```text
GET  /api/health        健康检查
GET  /api/records       获取全部吊装记录
POST /api/records       上传一条吊装记录
GET  /api/stats         质量统计汇总
GET  /api/ai/health     AI 是否可用（是否已配置 DeepSeek Key）
POST /api/ai            AI 对话/分析代理（转发 DeepSeek，Key 不出服务端）
```

## 真实 AI 能力（DeepSeek）

平台已接入 **DeepSeek 大模型**，核心决策类功能优先调用真实 AI，后端或网络不可用时自动回退到本地规则：

| 功能 | 模块 | 说明 |
| --- | --- | --- |
| **数字人施工指挥助手** | 智慧工地 | 以装配式技术专家身份实时问答，结合现场风速/温度判断能否吊装、解答规范要求等 |
| **图纸核对智能分析** | 装配慧检 | 在确定性逐项核对（实测⇄图纸）之上，由 AI 生成专业核对结论、超差原因分析与整改建议、规范要点 |
| **AI 作业票/整改闭环** | AI作业票 | 根据三确认、一复核、两到位和吊装控制参数，自动生成准吊判定、风险清单与整改复核要求 |
| **BHI 运维健康评估** | 智慧大屏 | DeepSeek 读取振动/裂缝/倾角/位移/温湿度传感数据，生成运维健康评估和闭环建议 |
| **整改派单与规范助手** | AI闭环中心 | 根据超差、吊装、节点、基层等问题生成整改工单、复检清单、规范依据、施工日志和班组评分 |

**密钥安全**：API Key 只存于服务端 `backend/ai-config.json`，并已加入 `.gitignore`，**不会**随前端代码或公开仓库泄露；前端只调用后端 `/api/ai` 代理。后端离线或未配置 Key 时，以上功能自动回退到内置应答，不影响其它功能。

**配置方法**：复制 `backend/ai-config.example.json` 为 `backend/ai-config.json`，填入自己的 DeepSeek Key（或设置环境变量 `DEEPSEEK_API_KEY`），重启后端即可。
> ⚠️ 若 Key 曾以明文形式分享过，建议在 DeepSeek 控制台**重置**后再正式使用。

## 五、演示建议（评委/答辩）

> 用 `启动平台.bat`（带后端 + 已配置 DeepSeek）演示，AI 为真实大模型应答；模块间点击即时切换、无刷新。

1. **智慧大屏** 总览项目态势与关键指标。
2. **装配慧检·录入**：填写双墙体数据 → 在“高度三点”输入相差 >10mm 的值并点空白处，触发**红色大框 + 警笛报警**；关键节点上传照片看 AI 检测动效。
3. **图纸 AI 核对**：选图纸编号点“识别设计值”→ 改一个实测值制造超差 → 点「**AI 自动核对**」，看确定性核对表 + **DeepSeek 生成的专业结论与整改建议**。
4. 切 **质量看板**：合格率趋势与质量分布；上传记录后实时刷新。
5. **智慧工地**：人机料法环实时数据、环境自动判定吊装条件，向 **数字人助手**提问“现在能否吊装？”（真实 DeepSeek 应答）。
6. **一码溯源**：输入构件 cid，查看数字身份证与工序留痕。

> 首次打开自动注入演示记录，便于看板与溯源展示；正式使用可上传真实数据。
